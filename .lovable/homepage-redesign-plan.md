# 首页交互设计优化方案 v2.0

> 基于当前项目代码深度分析，结合 Flux 极简美学方向，输出完整的首页重构方案。

---

## 一、现状分析与问题诊断

### 1.1 当前首页结构（自上而下）

| 序号 | 模块 | 组件 | 现状评估 |
|------|------|------|----------|
| ① | 页面标题 | 内联代码 | ✅ 简洁，头像+日期，无大问题 |
| ② | 测试按钮组 | 内联代码 | ❌ **必须移除** — 8个测试入口按钮占据大量首屏空间，严重破坏用户体验 |
| ③ | 断食状态卡片 | `FastingCard.tsx` | ⚠️ 功能完整但信息密度过高：圆环+阶段信息+AI提示+时间编辑+结束按钮全部堆叠在一张卡片内，视觉压迫感强 |
| ④ | 三餐时间轴 | `MealTimeline.tsx` | ✅ 右滑删除+引导弹动已实现，结构清晰 |
| ⑤ | AI 分析卡片 | `AISummaryCard.tsx` | ✅ 渐变玻璃质感良好，但与能量条缺乏视觉关联 |
| ⑥ | 动态能量条 | `NutrientEnergyBar.tsx` | ✅ 已完成毛玻璃+阶梯入场+数值滚动+光点动效 |
| ⑦ | 底部导航 | `BottomNavigation.tsx` | ⚠️ 中间拍照按钮突出但功能路径不够清晰 |

### 1.2 核心问题总结

1. **首屏效率低**：测试按钮组占据黄金位置，用户看不到核心内容
2. **信息层级混乱**：断食卡片内含5个功能区块，缺乏主次划分
3. **缺少"今日概览"感**：用户进入首页无法一眼获得今日健康状态的全局视图
4. **模块间缺乏叙事连贯性**：断食→三餐→AI分析→能量条，各模块独立存在，缺乏流动感
5. **体重曲线卡片（WeightCurveCard）已开发但未被首页使用**
6. **Index.tsx 已达308行**，需拆分

### 2.2 模块详细设计

---

#### ② 今日状态概览条（🆕 新组件：`DailyStatusBar`）

**设计目标**：让用户在首屏0.5秒内获取今日健康全貌

**视觉规格**：
- 水平排列4个胶囊状标签，超出屏幕可横向滑动
- 每个胶囊：`h-9 px-4 rounded-full bg-card shadow-card`
- 内含：彩色圆点(6px) + 标签文字 + 数值
- 胶囊间距：`gap-2.5`

**4个胶囊数据**：

| 胶囊 | 圆点颜色 | 标签 | 数值示例 | 数据源 |
|------|----------|------|----------|--------|
| 断食时长 | `--primary` | 断食 | 16:02 | fastingStore |
| 今日摄入 | `--warning` | 摄入 | 785 kcal | mealsData 计算 |
| 当前体重 | `--accent` | 体重 | 71.0 kg | weightStore |
| 记录进度 | `--lunch` | 记录 | 2/3 顿 | mealsData 计算 |

**入场动画**：
- 4个胶囊按 60ms 间隔从右侧滑入（`translateX(20px) → 0`），配合淡入
- 使用 `framer-motion` 的 `stagger` 编排

**交互**：
- 点击"断食"胶囊 → 平滑滚动至断食卡片
- 点击"摄入"胶囊 → 平滑滚动至 AI 洞察区
- 点击"体重"胶囊 → 平滑滚动至体重趋势卡片
- 点击"记录"胶囊 → 平滑滚动至三餐时间轴

#### ③ 断食核心卡片（重构 `FastingCard.tsx`）

**设计原则**：Less is More — 圆环即核心，其余按需展开

**精简后结构**：
```
┌─────────────────────────────────────┐
│  [Flux]           [补录断食] [📅]    │  ← 顶栏不变
├─────────────────────────────────────┤
│                                     │
│         ┌──── 圆环进度 ────┐         │
│         │   剩余时间        │         │
│         │   16:02:45       │         │
│         │   细胞自噬期      │         │
│         └─────────────────┘         │
│                                     │
│    🩸血糖  🔥脂肪  ⚡酮症  ✨自噬    │  ← 阶段图例（保持）
│                                     │
│  ┌─ 收起的阶段详情（可展开）─────┐    │  ← 🆕 折叠面板
│  │  当前：能量储存中              │    │
│  │  下一阶段：血糖平稳期 4h后    │    │
│  │  AI提示：刚享用完美餐...      │    │
│  └──────────────────────────────┘    │
│                                     │
│  开始 今天19:30    结束 明天11:30    │  ← 简化为单行展示（移除编辑按钮）
│                                     │
│  ┌──────── 结束断食 ────────┐        │
│  └──────────────────────────┘        │
└─────────────────────────────────────┘
```

**关键改动**：
1. **折叠阶段详情**：将"当前阶段信息"+"AI提示"合并为可展开/收起的折叠区域，默认收起，点击展开。减少首屏高度约120px
2. **简化时间显示**：开始/结束时间合并为单行，移除编辑铅笔按钮（编辑通过长按触发）
3. **圆环居中突出**：增加圆环上下 padding，让视觉焦点更聚焦

**折叠面板动效**：
- 使用 `framer-motion` 的 `AnimatePresence` + `layout` 实现丝滑高度过渡
- 展开时内容从上方淡入，收起时向上淡出
- 展开/收起按钮：一个极简的 `ChevronDown` 图标，展开时旋转180°

#### ⑤ AI 洞察区（合并重构：`AIInsightsSection`）

**设计目标**：将 AI 分析卡片与能量条合并为统一的"智能洞察"区块

**结构**：
```
┌─────────────────────────────────────┐
│  ✨ AI 综合餐饮分析                  │  ← 渐变背景头部
│                                     │
│  ┌── 今日摄入 ──┐ ┌── 餐食记录 ──┐  │
│  │   785 kcal   │ │    2 顿      │  │
│  └──────────────┘ └──────────────┘  │
│                                     │
│  ℹ️ 今日饮食结构分析：               │
│  今日热量控制良好...                  │
│                                     │
├─────── 分隔线（渐变淡出）────────────┤
│                                     │
│  ┌─ 营养摄入详情 ─────────────────┐  │  ← 能量条区域（融入同一容器）
│  │  脂肪    碳水    蛋白质   微量   │  │
│  │   24      180     52      68   │  │
│  │   g        g       g       %   │  │
│  │  ━━━     ━━━━━   ━━━━    ━━━  │  │
│  └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

**视觉处理**：
- 整体容器使用 AI 卡片的渐变背景，能量条区域使用 `bg-white/10 backdrop-blur-sm` 嵌套
- 移除能量条外部的独立毛玻璃容器，改为内嵌在AI卡片底部
- 分隔线使用 `bg-gradient-to-r from-transparent via-white/20 to-transparent h-[0.5px]`

**入场动画**：
- AI 卡片整体 `scale-in`（已有）
- 内部能量条保持阶梯式入场 + 数值滚动（已实现）
- 新增：AI 分析文字的打字机效果（逐字显示，30ms/字），营造 AI 实时分析的感知

#### ⑥ 体重趋势卡片（启用 `WeightCurveCard`）

**现状**：组件已开发完成但未在首页引用

**接入方式**：
- 直接在 AI 洞察区下方渲染 `<WeightCurveCard />`

**入场动画**：

#### ⑧ 底部导航优化

**现状问题**：中间拍照按钮点击后的流程不够直观

**优化方案**：
- 拍照按钮点击后，直接弹出 `PhotoMealModal`（已简化为一级选择餐时 + 拍照/相册按钮）
- 免费用户点击 → 先弹出 PaywallModal（已实现）
- 付费用户点击 → 直接弹出 PhotoMealModal

**视觉微调**：
- 拍照按钮增加微弱的脉冲呼吸动效：`animate-pulse-soft`，引导用户注意力
- 当有未记录的餐食时，按钮右上角显示一个小红点（`w-2.5 h-2.5 bg-destructive rounded-full`）

## 五、设计Token使用规范

所有组件必须使用 `index.css` 中定义的语义化 CSS 变量，禁止硬编码颜色值：

| 场景 | 正确用法 | 错误用法 |
|------|----------|----------|
| 卡片背景 | `bg-card` | `bg-white` |
| 主文字 | `text-foreground` | `text-gray-900` |
| 次要文字 | `text-muted-foreground` | `text-gray-500` |
| 早餐色 | `text-breakfast` / `bg-breakfast-light` | `text-orange-500` |
| 阴影 | `shadow-card` | `shadow-md` |
| 圆角 | `rounded-2xl` (1.25rem) | `rounded-[20px]` |
